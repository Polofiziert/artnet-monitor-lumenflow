## 1. Spec & “Luftschlösser” Assurance

**Goal:** Ensure the Rust Art-Net implementation and virtual scripts (spawn-virtual-network, virtual-console, virtual-node) stay aligned with the Art-Net 4 spec and we don’t build “Luftschlösser” (castles in the air).

**Actions:**

- **Canonical reference:** Treat the Art-Net 4 specification (Artistic Licence) as the single source of truth. All new OpCodes and packet handling must follow `.cursor/skills/spec-compliance/SKILL.md` (wire structs, zerocopy, hex tests).
- **Virtual scripts:** They already use `lumenflow_core` builders (ARTNET_SPEC_ASSESSMENT.md). Add/keep hex-encoded tests for any new or touched OpCodes; run `cargo test -p lumenflow_core` and chaos/network-simulation tests in CI.
- **Discovery / IP ranges:** Spec uses 2.x.x.x, 10.x.x.x, 127.x for directed broadcast. Many real products use 192.168.x.x. Plan (see Section 5) a Settings section for discovery/target IPs that is “spec-first” but allows optional non-spec ranges for real-world compatibility—document the trade-off and keep spec behaviour as default.
- **Regression:** On every change to artnet parsing or discovery, run existing spec and fuzz targets; no new `.unwrap()` in the hot path (per PROJECT_INSTRUCTIONS).
- **Backlog tests:** Every backlog item that involves behaviour or API changes has test requirements in Section 2 (column **Tests**) and in Section 6 (Test requirements). Implement the listed tests when implementing the item; see `docs/development/TESTING.md` for test setup and patterns.

## 4. UX/UI Notes (Research-Aligned)

- **Details on demand:** Channel details and device diagnostics follow “overview first, zoom and filter, then details on demand” (PROJECT_INSTRUCTIONS). Popovers should not change layout (B2).
- **Pro-Lab aesthetic:** Use design tokens from `.cursor/skills/apple-aesthetic-ui/SKILL.md` (Deep Obsidian, Cyber Teal, Safety Amber); Tailwind only; minimal clutter; tooltips and help reduce cognitive load.
- **Command palette:** Raycast/Arc-style search (T2) fits power users and reduces clicks; ⌘K is a familiar affordance.
- **Fixed-width status:** Prevents layout shift and respects reading order (B3).
- **Spec-first, compatibility option:** Discovery IP settings (S2) keep spec as default and document any extension for real-world devices.
- **Shared overlay primitive:** A single `FloatingPopover` (or OverlayPanel) component—anchor-based, fixed/absolute, optional arrow—used for channel detail (B2) and tooltips (H3, U1) keeps behaviour and Pro-Lab styling consistent and avoids layout reflow.
